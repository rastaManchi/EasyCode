Базовый URL

http://ваш-домен/api/v1/

Аутентификация

Для аутентификации используйте заголовок:

Authorization: Bearer ваш_токен

Эндпоинты

GET Получение всех постов

/api/v1/posts

Параметры: page (номер страницы), per_page (постов на страницу)

Пример: /api/v1/posts?page=1&per_page=10

GET Получение одного поста

/api/v1/posts/{id}

Пример: /api/v1/posts/1

POST Вход в систему

/api/v1/login

Тело запроса (JSON): 

    "email": "user@example.com",
    "password": "password123"

POST Создание поста (требует аутентификации)

/api/v1/posts/

Тело запроса (JSON):

"title": "Заголовок поста",
"content": "Содержание поста...",
"category_id": 1

PUT Обновление поста (требует аутентификации)

/api/v1/posts/{id}

Тело запроса (JSON):

"title": "Новый заголовок",
"content": "Новое содержание...",
"category_id": 2

DELETE Удаление поста (требует прав администратора)

/api/v1/posts/{id}

Пример: /api/v1/posts/1

Пример использования с JavaScript (Fetch API)

// Получение постов
fetch('/api/v1/posts')
    .then(response => response.json())
    .then(data => console.log(data));

// Создание поста
fetch('/api/v1/posts', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer ваш_токен'
    },
    body: JSON.stringify({
        title: 'Новый пост',
        content: 'Содержание...',
        category_id: 1
    })
})
    .then(response => response.json())
    .then(data => console.log(data));

Коды ответов

  • 200 – Успешный запрос
  • 201 – Успешно создан
  • 400 – Ошибка в запросе
  • 401 – Требуется аутентификация
  • 403 – Недостаточно прав
  • 404 – Не найдено
  • 500 – Ошибка сервера
